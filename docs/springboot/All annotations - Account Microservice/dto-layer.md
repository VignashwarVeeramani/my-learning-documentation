# DTO (Data Transfer Object) Layer: Annotations

This document details the annotations used in the Data Transfer Object (DTO) classes. DTOs are plain Java objects used to transfer data between different layers of the application, especially between the controller and the client. They help to decouple the API contract from the internal data model (entities).

## 1. Lombok Annotations

These annotations from Project Lombok are used to significantly reduce boilerplate code like getters, setters, and constructors.

### `@Data`
A convenient shortcut annotation that bundles the features of `@ToString`, `@EqualsAndHashCode`, `@Getter`, `@Setter`, and `@RequiredArgsConstructor` together. It generates all the standard methods for a typical data-holding class.

**Sample Code (`CustomerDto.java`):**
```java
import lombok.Data;

@Data
public class CustomerDto {
    private String name;
    private String email;
    // ... other fields
}
```

### `@AllArgsConstructor`
This annotation generates a constructor with a parameter for every field in the class.

**Sample Code (`ResponseDto.java`):**
```java
import lombok.AllArgsConstructor;
import lombok.Data;

@Data @AllArgsConstructor
public class ResponseDto {
    private String statusCode;
    private String statusMsg;
}
```

## 2. Jakarta Bean Validation Annotations (from `jakarta.validation.constraints.*`)

These annotations are used to declare validation rules for the fields of the DTOs. When an object is annotated with `@Valid` in the controller, Spring automatically applies these validation rules.

### `@NotEmpty`
Asserts that the annotated element is not `null` and not empty.

**Sample Code (`AccountsDto.java`):**
```java
import jakarta.validation.constraints.NotEmpty;

public class AccountsDto {
    @NotEmpty(message = "AccountNumber can not be a null or empty")
    private Long accountNumber;
}
```

### `@Pattern`
Asserts that the annotated string matches the specified regular expression.

**Sample Code (`CustomerDto.java`):**
```java
import jakarta.validation.constraints.Pattern;

public class CustomerDto {
    @Pattern(regexp = "(^$|[0-9]{10})", message = "Mobile number must be 10 digits")
    private String mobileNumber;
}
```

### `@Size`
Asserts that the annotated element's size is between the specified `min` and `max` boundaries (inclusive).

**Sample Code (`CustomerDto.java`):**
```java
import jakarta.validation.constraints.Size;

public class CustomerDto {
    @Size(min = 5, max = 30, message = "The length of the customer name should be between 5 and 30")
    private String name;
}
```

### `@Email`
Asserts that the annotated string is a well-formed email address.

**Sample Code (`CustomerDto.java`):**
```java
import jakarta.validation.constraints.Email;

public class CustomerDto {
    @Email(message = "Email address should be a valid value")
    private String email;
}
```

## 3. OpenAPI (Swagger) Annotations (from `io.swagger.v3.oas.annotations.media.*`)

These annotations are used to provide metadata for the API documentation generated by Swagger/OpenAPI.

### `@Schema`
Allows you to provide metadata about your DTOs and their properties, which is then used to generate rich and descriptive API documentation.

*   **On a class**: Provides a name and a description for the DTO itself in the OpenAPI schema.
*   **On a field**: Provides a description and an example value for a specific property.

**Sample Code (`CustomerDto.java`):**
```java
import io.swagger.v3.oas.annotations.media.Schema;

@Schema(
        name = "Customer",
        description = "Schema to hold Customer and Account information"
)
public class CustomerDto {

    @Schema(
            description = "Name of the customer", example = "Eazy Bytes"
    )
    private String name;

    @Schema(
            description = "Email address of the customer", example = "tutor@eazybytes.com"
    )
    private String email;
    
    // ...
}
```
