# Accounts Microservice: A Deep Dive into Annotations

This section provides a comprehensive, layer-by-layer breakdown of all the annotations used in the **Accounts Microservice**. 

Each page is dedicated to a specific architectural layer, offering detailed explanations and code samples for every annotation used within that layer.

## Architectural Layers

Navigate through the sidebar or use the links below to explore the annotations in each layer:

- **[Application Layer](./application-layer.md)**: The main entry point of the Spring Boot application.
- **[Controller Layer](./controller-layer.md)**: Handles incoming REST API requests.
- **[Service Layer](./service-layer.md)**: Contains the core business logic.
- **[Repository Layer](./repository-layer.md)**: Manages data access and database operations.
- **[Entity Layer](./entity-layer.md)**: Defines the database schema through JPA entities.
- **[DTO Layer](./dto-layer.md)**: Data Transfer Objects for API communication.
- **[Exception Layer](./exception-layer.md)**: Global exception handling.
- **[Mapper Layer](./mapper-layer.md)**: Handles object-to-object conversions.
- **[Audit Layer](./audit-layer.md)**: Manages auditing and tracking entity changes.
